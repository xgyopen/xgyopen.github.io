质检机器人

畸变校正

一般给畸变建模有3种：

  • 径向畸变（radial distortion）：k1,k2,k3,k4,k5,k6
  • 切向畸变（tangential distortion）：p1,p2
  • 薄透镜畸变（thin prism distortion）：s1,s2,s3,s4

一般是先选择合适的硬件，再从软件层面去优化。（长焦镜头畸变小，远心镜头几乎没有畸变）
用张正友的相机标定的方法（1999年PAMI）去估计畸变效果就很好。

实验实测下来，一般考虑到2阶径向畸变结果比较好。更复杂的模型不仅没有帮助，而且会导致数值不稳定。

手眼标定

手眼标定建立的是机械臂（手）与相机（眼）之间的转换关系（4×4的转换矩阵），本质是解CX=XD的问题，C与相机相关，D与机械臂相关。

主流的5篇论文效果差不多，但1989年Tsai的效果最稳定。

  • Tsai_1989_A New Technique for Fully Autonomous and Efficient 3D Robotics Hand Eye Calibration
  • Navy_1994_Robot Sensor Calibration：Solving AX = XB on the Euclidean Group
  • Inria_1995_Hand-Eye Calibration
  • DualQ_1999_Hand-Eye Calibration Using Dual Quaternions => Halcon采用
  • Kron_2001_Robot Hand-Eye Calibration Using Structure-from-Motion

双目用棋盘格，点云用凸台标定块（2019年1月，VisionPro9.5中有基于结构光的手眼标定的例程）。

  • 棋盘格：机械臂带着相机/标定板走9个姿态（交叉走位），做相机标定、手眼标定即可
  • 标定块：利用凸台上先验的8个角点，导入凸台CAD模型用ICP做配准，获取8个点坐标，用LM法即可求出不错的结果

图像处理定位物体（基于OpenCV）

机械臂抓取分为2D和3D两种情况（2019年3月的Vision China机器视觉展）：

  • 2D：解决不了工件倾斜的问题
  • 3D：
    • 基于双目特征（定位、引导、装配）：流程是固定的，重在特征的选取，精度最高做到0.5mm左右
    • 基于点云配准（无序抓取）：重在点云分割、点云配准，精度做到mm级

2D特征必须准确且可靠（不会被3D影响，如透视收缩）

稳定的2D特征（能承受3D的不确定，避免“圆角”等不稳定的特征）：

  1. 尖锐、平直的边缘
  2. 圆
  3. 具有旋转、缩放、平移不变性的独特特征

机械手编程（物体抓取和放置）

做过丹麦UR机器人的，大概思路：

  1. 模板位，对机器人做示教抓取（路点/过渡点）
  2. 偏移位，视觉系统返回偏移给机器人，机器人偏移坐标系实现抓取

做项目时和别的公司配合过KUKA和ABB的，思路相同。

机器视觉行业应用经验、对机械手系统集成有清晰的概念

关于双目特征，做过实际项目，上海中车瑞伯德地铁车窗装配，1.6m*1.2m的视野，精度做到1mm。

感悟：

  • 先硬件后软件（长焦镜头畸变小）
  • 通过打光提高稳定性（波动2个像素以内）
  • 标定的9个姿态工作距离相同

关于点云配准，正在研究尝试用深度学习的方法解决无序抓取的问题，大体思路：
（CVPR2017的3DMatch、CVPR2017的PointNet、NIPS2017的PointNet++、CVPR2018的SGPN）

  1. 扫描一个工件/导入CAD，用最小包围盒OBB按照物理规律生成数据，做数据增强。
  2. 用3DMatch训练特征描述子。
  3. 对真实数据，用SGPN做点云的实例分割。
  4. 对分割后的数据，随机选取关键点，用3DMatch计算特征嵌入向量，选择特征空间内距离相近的关键点，结合RANSAC做点云配准/ICP。

熟悉深度学习框架

对MXNet比较熟悉，参加过天池的比赛，56/2972。

一些观点

传统图像处理VS深度学习

传统图像处理：像素维
深度学习：特征维

张氏相机标定细节？

相机标定IO：
输入：标定板物理坐标系中的角点坐标（姿态数×角点数×3）、图像像素坐标系中的角点坐标（姿态数×角点数×2）
输出：相机内参、畸变系数、相机外参

  1. 估算单应性矩阵H（封闭解）
  2. 计算相机内参
  3. 计算径向畸变k1,k2,相机外参
  4. 最大似然估计

Tsai手眼标定细节？

手眼标定IO：
输入：机械臂两个姿态之间的转换关系（示教器读取）、相机两个姿态之间的转换关系（相机外参）
输出：手眼矩阵（4×4）

网上有苏黎世理工的Matlab开源实现，移植到python下和C++下就可以了。

两步法：先算R、再算T（大量运用最小二乘法）

  1. 利用罗德里格斯变换将旋转矩阵转换为旋转向量
  2. 向量归一化
  3. 修正的罗德里格斯参数表示姿态变化
  4. 计算初始旋转向量
  5. 计算旋转向量
  6. 计算旋转矩阵
  7. 计算平移向量

参考：经典手眼标定算法之Tsai-Lenz的OpenCV实现

偏移计算细节

一般做法是将其看作优化问题，将伪逆作为初始值，只考虑仿射变换中的旋转、平移，用LM法优化

整个流程（EIH/ETH）？

两种情况，大体相近：

分类 相机关系 所求关系
眼在手外 相机固定 Camera3D与RobotBase3D的关系
眼在手上 相机移动 Camera3D与Hand3D的关系

眼在手上

坐标系变换关系为：CalPlate3D → Camera3D → Hand3D → RobotBase3D

  1. 相机标定
    • 输入：标定板物理坐标系中的角点、图像像素坐标系中的角点
    • 输出：相机内参、畸变系数、相机外参
  2. 手眼标定
    • 输入：机器人两个姿态间的转换矩阵（后一个姿态的逆×前一个姿态）、相机两个姿态间的转换矩阵（后一个外参×前一个外参的逆）
    • 输出：手->眼 的逆
  3. 三角测量（2D点要纠正畸变）
    • 输入：左右相机对应的同一特征点的2D坐标
    • 输出：该特征点在基坐标系下的3D点坐标
  4. 计算偏移
    • 输入：模板位至少4个特征点、偏移位至少4个特征点
    • 输出：模板位到偏移位的转换关系（旋转+平移）

眼在手外

坐标系变换关系为：Hand3D → RobotBase3D → Camera3D → CalPlate3D

  1. 手眼标定
    • 输入：机器人两个姿态间的转换矩阵（后一个姿态×前一个姿态的逆）、相机两个姿态间的转换矩阵（后一个外参×前一个外参的逆）
    • 输出：眼->基坐标系

坐标系转换

  1. 左右相机通过特征提取，得到Raw2D下的特征点坐标
  2. 通过相机内参，将特征点转到Camera2D下
  3. 把未知高度作为未知数，将特征点转到Camera3D下
  4. 通过手眼矩阵，将特征点转到Hand3D下
  5. 通过示教器读取的机械手位姿，将特征点转到RobotBase3D下
